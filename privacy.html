Privacy Policy

Foliow desktop application · Last updated: 2026-05-20

1. Data Collection

Foliow is a fully offline desktop application. All user-created data (works, characters, worldbuilding, manuscripts, publisher contracts, and writing progress) is stored only on the user's own computer. Foliow does not operate any server and does not transmit user content to any third party by default.

2. Dropbox Integration (Optional)

If the user chooses to connect a Dropbox account, Foliow uses Dropbox solely to read and write backup archives (.zip files) inside the app-scoped Apps/Foliow/ folder. Foliow does not access any other files in the user's Dropbox account.

The Dropbox connection is initiated by the user from within the app and can be disconnected at any time. Disconnecting removes the locally stored authentication tokens.

3. Authentication Tokens

OAuth tokens received from Dropbox are stored on the user's local computer using the operating system's secure storage facility (Electron safeStorage, backed by the OS keychain on macOS and DPAPI on Windows). Tokens are never transmitted to any server operated by the developer.

4. Auto-Update

Foliow checks for new releases by querying public GitHub release metadata. No personal information is transmitted in this request beyond standard HTTP headers required by the protocol.

5. License Verification

License keys are verified entirely offline using Ed25519 cryptographic signatures. No license activation data is sent to any server.

6. Third Parties

Foliow does not use analytics, telemetry, advertising SDKs, or any third-party tracking services. The only optional external service is Dropbox, as described in section 2.

7. Changes to This Policy

If this policy is updated, the "Last updated" date at the top of this page will reflect the most recent revision. Material changes that affect how user data is handled will be communicated through the application's release notes.
